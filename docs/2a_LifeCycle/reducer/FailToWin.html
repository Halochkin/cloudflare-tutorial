Basically the wrapSuccessError will: 1. take in both async/sync functions, with arguments, 2. invoke it, and 3. return an object {success, error} with the result. The asyncErrorRace will run any function against a given list of arguments. The function receives its arguments as an object with an array of string properties. The function returns an object with {success, error} values. * If an error is thrown during the execution, then the error value will be populated. * If no error occurs during the execution, then the success value will be filled. The asyncErrorRace can also run async functions. If the function returns a Promise, then when that Promise resolves, it will resolve the {success} value, and if that Promise fails, it will resolve the {error} value. The function will not run until all arguments are resolved * To allow a function to run even though one or more arguments are not resolved, mark the optional arguments with a '*' prefix. If none of the arguments given in are Promises, the function will run sync/immediately.
